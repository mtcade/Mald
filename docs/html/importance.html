Interface to calculate the MALD Importance and W Statistics given a PredictionModel, either your own or created through maldImportance.superBasicNetworks; see <no title>.

If you do wish to use a maldImportance.superBasicNetworks.SimpleNN then you can use the maldImportance.nnImportance (<no title>) module for simplicity.

maldImportance.importance.importancesFromModel(model: PredictionModel, X: ndarray | DataFrame, Xk: ndarray | DataFrame, y: ndarray | Series, local_grad_method: Literal['auto_diff', 'bandwidth'] = 'auto_diff', fit: bool = True, bandwidth: float | None = None, exponent: float = 2.0, drop_first: bool = True, verbose: int = 0) → ndarray¶
Parameters:

  • model (PredictionModel) – Predictor which can use autodifferentiation

  • X (np.ndarray|pd.DataFrame) – Explanatory data

  • Xk (np.ndarray|pd.DataFrame) – Knockoff explanatory data

  • y (np.ndarray|pd.Series) – Outcome data

  • local_grad_method (Literal['auto_diff','bandwidth']) – Method of MALD. Defaults to ‘auto_diff’ for exact autodifferentiation. ‘bandwidth’ uses the bandwidth approximation when auto differentiation is not available.

  • fit (bool) – Whether to fit model. Set to False if you have already trained it to your satisfaction on the combined explanatory and knockoff data together.

  • bandwidth (float|None) – Width if local_grad_method = ‘bandwidth’

  • exponent (float) – Power to take of each MALD value. 1.0 and 2.0 both work reasonably well.

  • drop_first (bool) – How to handle one-hot-encoding categorical variables. If True the number of associated columns is the number of categories minus 1.

  • verbose (int) – How much to print out, for mostly for debugging.

Returns:

Array of importances, with length equal to twice the width of X

Return type:

np.ndarray

Takes an initialized PredictionModel, likely fits it, and gets the MALD importances for each X and Xk variable
